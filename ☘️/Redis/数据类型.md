# 数据类型
> 相关笔记：[[Redis|Redis 知识总结]]


注意，Redis服务器中存储数据的时候只是按照二进制形式进行存储，包括对字符串进行操作时都是按照字节维度进行操作。因此，对中文以及国外的语言数据进行操作时要注意编码方式。

Redis中Key是固定字符串形式，但Value可以是很多种类型的

![image](image-20260408200947-82yysy0.png)

除了有序集合外其他的都不陌生，这里讲解下有序集合。

- 有序集合：相当于除了存储member外（如图上关羽、刘备、张飞），还存储了一个score（我们叫他权重 / 分数），依靠权重大小来分配优先权。

Redis之所以相对于MySQL更快，有一部分是在实现这些数据结构时，会在源码上面进行了一系列优化，达到节省时间 / 空间的效果。

背后的实现不一定是一个标准的哈希表，再不同的场景下可能会采用不同的实现策略，但是都能达到更好的效果。故同一种数据类型，会根据特定的场景来进行不同的优化，通常有：

|数据结构|内部编码|
| ----------| -----------------------|
|String|raw<br />int<br />embstr|
|hash|hashtable<br />ziplist|
|list|linkedlist<br />ziplist<br />**quicklist**|
|set|hashtable<br />inset|
|zset|**skiplist**<br />ziplist|

- raw：最基本的字符串，底层就是持有一个char数组[C++] / byte数组[Java]

  注意：C++的char数组是一个字节，对应Java中的byte，而Java的char是两个字节的
- embstr：这是Redis对短字符串做的特殊优化
- hashtable：标准的哈希表
- ziplist：压缩列表，能节省空间

  为啥要用压缩？ 👉 Redis上如果有很多的Key，且Key的value都是hash，但hash都不大，就尽量去压缩节省内存空间
- linkedlist：链表
- quicklist：同时兼顾了linkedlist和ziplist的优点，quicklist本身就是链表，每个元素又是ziplist，存更多的信息
- intset：集合中存的都是int整数
- skiplist：跳表。正常链表下，每个节点都存有next，next指向下一个节点，跳表也是链表，不同于普通链表，它每个节点上有多个指针域，Redis通过巧妙搭配这些指针指向，能做到从跳表上查询元素的时间复杂度是O(logN)。

要是想查看Key对应Value的实际编码方式，可通过命令`object encoding key`查看
