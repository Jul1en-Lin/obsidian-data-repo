# socket 套接字

socket 提供了应用层与传输层之间通信的编程接口，在操作系统中被抽象为类似文件的对象，通过一组系统调用如`send()，receive（），accept（）`等方法使用，用来传输数据

socket这个词可以有多种含义。例如网络编程套接字 => 传输层中的各种API接口、或抽象为“特殊的文件”

而socket对象可以指 ☞ 1. 表示网卡硬件设备的抽象 2.文件

文件有广泛的含义：如控制台、打印机、硬盘、网卡....，~~这些都是由操作系统提供了socket文件的ap~~i，这些都是系统由文件描述符来抽象

Socket是专门针对网络编程而抽象出的一种文件描述符，它有一组自己的API（`send()、receive()、connect()`​），Java再对socket进行封装成了对象，通过`socket.getInputStream() \ socket.getOutputStream`来读写数据流	

‍

传输层主要涉及两大协议

1. UDP —— 无连接，不可靠传输，全双工，面向数据报
2. TCP —— 有连接，可靠传输，全双工，面向字节流

# 无连接 / 有连接

💌注意：计算机中连接是一种抽象的概念，比如我和小美结婚了，拿了结婚证，这就是一种“连接”

               而链接而是`link`的意思，比如快捷方式，某个网址地址.....

- 无连接

  不保存对端的信息（IP与端口号），类似于发短信

- 有连接

  保存对端的信息（IP、端口号），类似于打电话

# 不可靠传输 / 可靠传输

- 不可靠传输

  不担保数据能一定传到对端，大概率会有网络丢包或数据乱序的情况，但传输效率高
- 可靠传输

  传输过程检验数据是否发到对端，需要花费校验时间，效率不如不可靠传输

# 全双工 / 半双工

- 全双工

  同一时间通信双方都能同时接收与发送数据，类似微信 / qq聊天 / 打电话（双方都有发送缓冲区+接收缓冲区）

  ```java
        ┌───────────────────────┐
        │        主机 A         │
        │ ┌───────────────┐     │
        │ │ 发送缓冲区 →→→ │────────────┐
        │ │               │     │      │
        │ │ ←←← 接收缓冲区 │     │      │
        │ └───────────────┘     │      │
        └───────────────────────┘      │
                                        │ 双向同时通信
        ┌───────────────────────┐      │
        │        主机 B         │      │
        │ ┌───────────────┐     │      │
        │ │ 发送缓冲区 →→→ │────────────┘
        │ │               │
        │ │ ←←← 接收缓冲区 │
        │ └───────────────┘
        └───────────────────────┘

  ```
- 半双工

  同一时间，只能单向传输数据，一方必须发完另一方才是再发，类似对讲机（只有单向通道）

  ```java
        ┌───────────┐
        │   主机 A  │
        │   (说话)  │
        └─────┬─────┘
              │
              ▼
           单向通道
              ▲
              │
        ┌─────┴─────┐
        │   主机 B  │
        │   (等待)  │
        └───────────┘
  ```

## UDP服务器

以回显服务器为例子——客户端发什么，服务器原封不动返回

构造服务器需要一个端口号，需要手动设置（1024 ~ 65535）

```java
    private DatagramSocket socket = null;
    public int port;
    public UdpEchoServer(int port) throws SocketException {
        //手动指定服务器端口号
        socket = new DatagramSocket(port);
        this.port = port;
    }
```

服务器需要运行，一般是随时待命7*24小时的，所以要在`while（true）`中循环处理客户端不断来的请求

![image](image-20250925144407-byw2e0q.png)

服务器的执行逻辑分为三大部分

1. 收到请求

   DatagramPacket接收客户端发来的请求

   ```java
   DatagramPacket requestPacket = new DatagramPacket(new byte[1024],1024);//输出型参数
   socket.receive(requestPacket);//等待阻塞
   ```
2. 根据请求计算响应

   接收到了数据报，并进行解析，单独写process方法可以解耦合

   由于数据报中保存的是二进制数据，但是我们这里的回显服务器传输的是字符串类型，所以解析需要将二进制转换成字符串

   ```java
   //将二进制数据转换为字符串
   String request = new String(requestPacket.getData(),0,requestPacket.getLength());
   //2. 解析请求
   String response = process(request);
   ```
3. 把响应返回给客户端`send()`

   数据报收到与发送都是二进制的形式，故将response字符串转换成二进制格式

   ```java
   //3. 发送响应，再打包一个DatagramPacket发回
   DatagramPacket responsePacket = new DatagramPacket(response.getBytes(),
        /*response.length() error*/
        //输入的不是字符串的长度，而是字节数组的长度，否则长度变小，导致输出内容不完整
        response.getBytes().length,
        /*requestPacket.getAddress(),requestPacket.getPort() error*/
        requestPacket.getSocketAddress());//getSocketAddress就拿到了客户端IP和端口
   socket.send(responsePacket);
   ```

   由于UDP无连接的协议特性，所以是不会保存对端信息的，此处只是在设置数据要发给“谁”

完整代码

```java
public class UdpEchoServer {
    private DatagramSocket socket = null;
    public int port;

    public UdpEchoServer(int port) throws SocketException {
        //手动指定服务器端口号
        socket = new DatagramSocket(port);
        this.port = port;
    }

    public void start() throws IOException {
        System.out.printf("Server start! ");
        //7*24hs运行
        while (true) {
            //1. 收到请求
            /*byte[] bytes = new byte[1024];
            DatagramPacket requestPacket = new DatagramPacket(bytes, bytes.length);*/
            DatagramPacket requestPacket = new DatagramPacket(new byte[1024],1024);//输出型参数
            socket.receive(requestPacket);//等待阻塞
            //将二进制数据转换为字符串
            String request = new String(requestPacket.getData(),0,requestPacket.getLength());
            //2. 解析请求
            String response = process(request);
            //3. 发送响应
            DatagramPacket responsePacket = new DatagramPacket(response.getBytes(),
                    /*response.length() error*/
                    //输入的不是字符串的长度，而是字节数组的长度，否则长度变小，导致输出内容不完整
                    response.getBytes().length,
                    /*requestPacket.getAddress(),requestPacket.getPort() error*/
                    requestPacket.getSocketAddress());//包含客户端IP和端口
            socket.send(responsePacket);
            //4. 打印日志
            System.out.printf("[%s:%d] req: %s,resp: %s\n",
                    requestPacket.getAddress().toString(),responsePacket.getPort(),
                    request,response);
        }
    }

    //暂时是回显服务器，直接把request 当 response 返回
    //以后解析操作只需更改process方法即可
    public String process(String request) {
        return request;
    }

    public static void main(String[] args) throws IOException {
        //如果跟别人的端口号重复会报错
        UdpEchoServer server = new UdpEchoServer(9191);
        server.start();
    }

}
```

## UDP客户端

与服务端不同，构造客户端时不需要手动设置端口号，系统会自动分配，

那客户端怎么知道服务器在哪？——需要程序员告诉客户端，手动输入IP与端口号则是为了以后发送数据报时需要，也可以在`start()`方法才传入IP与端口号，这个不影响

```java
    private DatagramSocket socket = null;
    private String serverIP;
    private int serverPort;
    public UdpEchoClient(String serverIP,int serverPort) throws SocketException {
        //客户端端口由系统自动分配，无需手动配置
        socket = new DatagramSocket();
        this.serverIP = serverIP;
        this.serverPort = serverPort;
    }
	// 这种形式也是可以的
	public UdpEchoClient() throws SocketException {
        //客户端端口由系统自动分配，无需手动配置
        socket = new DatagramSocket();
    }
```

客户端执行时的逻辑

1. 读取用户输入并打包数据，发给客户端

   ```java
           // 1. 读取用户请求
           String request = scanner.next();
           if (request.equals("exit")) break;
           //2. 发送请求，将用户输入的字符串构造成UDP数据报，进行发送
           DatagramPacket requestPacket = new DatagramPacket(request.getBytes(),
                   /*request.length() error*/
                   //不是字符串的长度，而是字符串转换为字节后的长度，否则传输的长度有误，导致传输内容不完整
                   request.getBytes().length,
   ```

2. 从服务器读取响应并解析

   ```java

           System.out.println(response);
   ```

```java
public class UdpEchoClient {
    private DatagramSocket socket = null;
    private String serverIP;
    private int serverPort;
    public UdpEchoClient(String serverIP,int serverPort) throws SocketException {
        //客户端端口由系统自动分配，无需手动配置
        socket = new DatagramSocket();
        this.serverIP = serverIP;
        this.serverPort = serverPort;
    }

    //需要手动输入客户端的IP和端口
    public void start() throws IOException {
        System.out.println("Client start! ");
        Scanner scanner = new Scanner(System.in);
        while (true) {
            System.out.print("-> ");
            // 1. 读取用户请求
            String request = scanner.next();
            if (request.equals("exit")) break;
            //2. 发送请求，将用户输入的字符串构造成UDP数据报，进行发送
            DatagramPacket requestPacket = new DatagramPacket(request.getBytes(),
                    /*request.length() error*/
                    //不是字符串的长度，而是字符串转换为字节后的长度，否则传输的长度有误，导致传输内容不完整
                    request.getBytes().length,
                    InetAddress.getByName(this.serverIP),this.serverPort);
            socket.send(requestPacket);
            //3. 从服务器读取响应resp
            DatagramPacket responsePacket = new DatagramPacket(new byte[1024],1024);//输出型参数
            socket.receive(responsePacket);
            String response = new String(responsePacket.getData(),0,responsePacket.getLength());
            System.out.println(response);
        }
    }

    public static void main(String[] args) throws IOException {
        UdpEchoClient client = new UdpEchoClient("127.0.0.1", 9191);
        client.start();
    }
}
```
