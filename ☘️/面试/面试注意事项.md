# 面试注意事项
> 相关笔记：[[面试|面试 知识总结]]


1. 面试过程中保持姿态open端正
2. 说话可以慢一点，语气缓和一些
3. 面试的时候拿纸和笔
4. 每次面试都录音复盘

5. 诚实比完美更重要

   如果被问到不会的问题，直接说：

   > "这个我当时没深入研究，如果要做的话我会先查资料/看文档。"
   >

   比瞎编要好。

6. 主动暴露问题

   面试时可以主动说：

   > 这个项目还有挺多可以优化的地方，比如硬编码路径，后面学了Springboot之后发现可以写在配置文件中，根据不同场景来完成不同的配置。当时主要是快速验证想法。
   >

   这展示你的工程意识。

7. 准备改进方案

   对每个问题，准备"如果给我时间我怎么改"：

   - 硬编码路径 → 配置化 + 环境变量
   - 缺少缓存 → Redis 三级缓存
   - 单点故障 → Docker Compose 容器化

8. 强调学到了什么

   即使项目简单，也要强调收获：

   - 理解了倒排索引原理
   - 学会了 FAISS 向量检索
   - 意识到工程化（配置管理、错误处理）的重要性

---

遵循 STAR 原则

- S—Situation：事情是在什么情况下发生；
- T—Task：你是如何明确你的任务的；
- A—Action：针对这样的情况分析，你采用了什么行动方式；
- R—Result：结果怎样，在这样的情况下你学习到了什么。
