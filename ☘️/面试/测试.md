# 测试人员和开发人员产生争执了怎么办

以沟通为主，测试人员应具备良好的沟通能力，这在团队中是非常重要的

- 秉承批判性思维，先以自身出发：

  - 是不是自己的 bug 提交报告写的不够规范，不够清晰
  - 不光能提出 bug，最好还能提出解决方案的建议
  - 适当得体的语言能使得对方更能接受
  - 若讨论出现对立意见，不要上升情绪，保持讨论对事不对人的原则
  - 同时反思自己，对他人的建议不排斥并且批判性地吸收经验

- 从开发人员的角度思考，这个bug是否是需要提出来的，自己在提bug时是否是误操作导致的

- 再从用户角度出发，反问自己：“如果你是用户，你能够接受这样的问题吗”
