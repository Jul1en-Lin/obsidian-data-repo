## 概述

位图 bitmap 的核心思路是用一个 bit 表示一种状态，通常 `1` 表示存在 / 持有，`0` 表示不存在 / 不持有。它适合表示大量布尔状态，比如某个数字是否出现过、某个槽位是否属于某个节点。

![[assets/bitmap-concept.png|675]]

## 基本思路

如果要表示很多对象是否存在，普通数组可能会浪费空间；位图会把每个对象映射到一个下标，然后用这个下标对应的 bit 来表示状态。

比如下标 `3` 对应的 bit 是 `1`，就表示编号为 `3` 的对象存在；如果是 `0`，就表示不存在。判断是否存在时，看对应 bit 是不是 `1`；添加一个对象时，把对应 bit 置为 `1`。

## Redis 集群里的位图

Redis Cluster 固定有 `16384` 个槽位，每个节点需要告诉其他节点自己负责哪些槽位。这里就可以用位图表示：第 `i` 位为 `1`，表示当前节点持有第 `i` 个槽位；第 `i` 位为 `0`，表示当前节点不持有这个槽位。

```txt
16384 bit = 2048 byte = 2KB
```

所以一个节点只需要大约 `2KB` 的位图信息，就能表示自己负责哪些槽位。这个数据会随着节点心跳等信息一起传播，所以 Redis 选择 `16384` 个槽位也是在均衡性和通信开销之间做了折中。

## 注意事项

位图本身只能表示状态，不直接保存对象内容。如果要知道具体对象是什么，还需要结合下标映射规则一起理解。

如果一个位置需要保存复杂数据，就不适合只用一个 bit 表示。
