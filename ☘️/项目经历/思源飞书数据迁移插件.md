# 思源飞书数据迁移插件
> 相关笔记：[[项目经历|项目经历 知识总结]]


本项目基于 [siyuan-plugin-export-to-feishu](https://github.com/NormanZyq/siyuan-plugin-export-to-feishu) fork 开发，在模板基础上增添了脚本一键获取Access Token、Access Token 刷新过期时间的功能，省略了获取飞书 Access Token 的繁琐过程

PR：[https://github.com/NormanZyq/siyuan-plugin-export-to-feishu/pull/2](https://github.com/NormanZyq/siyuan-plugin-export-to-feishu/pull/2)

‍

面试 STAR 故事版：

S: 我在使用一个开源思源笔记插件（将笔记导出到飞书文档）时发现，原项目要求用户手动去飞书 API 调试台获得 user_access_token / tenant_access_token。我在使用 tenant_access_token 调用过程中文档并不能在我的飞书看到，但文档确实是导出来了。切换 user_access_token 就没有这种现象。

T: 我就去查，查到因为 tenant_access_token 是以企业应用身份创建文档，不会出现在个人空间中。而且我在获取 user_access_token 还要拿到 OAuth 授权，过程非常繁琐，我看拿 OAuth 授权的流程还是比较固定的，只要拿到`APP_ID`​和`APP_SECRET`就可以了，就想着能不能拓展一下，让上手难度没那么高

A: 我做了三件事：第一，调研了飞书 OAuth2.0 授权流程，写了一个 Python 脚本启动本地回调服务器，自动完成授权并输出 user_access_token；第二，实现了 refresh token 脚本，避免用户反复授权；第三，在 README中补充说明，详细解释两种 token 的区别和适用场景。

R: 这些改进让原本需要手动操作多步的 token 获取流程变成了一键执行，极大降低了非技术用户的使用门槛。我向上游仓库提交了 PR，目前已被 open。
