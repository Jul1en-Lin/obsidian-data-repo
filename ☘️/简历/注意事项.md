# 面试注意事项—面试前

![image](assets/image-20260401231422-bd627cc.png)![image](assets/image-20260401231437-wi308c2.png)

- 提前了解面试公司的规模、业务、产品、岗位等
- 简历最好是彩印的，有条件可以加塑封！！！！！
- 调试摄像头

- 带纸质版简历（最好彩印塑封）

- 见到面试官打招呼，年轻的喊“学长 / 学姐”，看到年龄较大的喊“老师”

- **<u>==准备一套自己的自我介绍（背！）~，且语言简洁明了==</u>**

  ![image](assets/image-20260401231543-udas0jy.png)

  自我介绍的内容：

  1. 个人基本信息：姓名，学校，专业。eg: 学长您好，我是一名来自五邑大学计算机专业的28届学生
  2. 求职意向或目标
  3. 个人技术背景（即：你会哪些）？

- 避免答非所问，回答尽量做到内容连贯，条理清楚（可以跟面试官沟通，准备一点时间来组织语言的）
- 不要"展开回答"，避免弄巧成拙，面试官问什么你就回答什么，展开回答可能会导致面试官怀疑你抓不住重点

![image](assets/image-20260401223045-e9f15pu.png)

- 谈薪资

  ![image](assets/image-20260401223014-sm4v9w5.png)
- 谈实习时间：问多久能够到岗？

  随时可以~！
- 谈工作地点：如果你要去一个比较远的分部你愿意去吗？

  当然是可以接受的，我还年轻我希望能够出去闯一闯，时间和地点对我来说不是问题

- 谈个人职业规划（比较重点）

  一般来说比较难谈好，毕竟对行业并不了解，但可以从短期规划入手（如未来的3~5年）

  eg: 未来的三到五年，我希望我在技术岗上沉淀，好好学习，积累经验，结合公司的业务来发展自己，慢慢提升。能够有能力独立负责某一块业务，能够带几个人，做一个小组负责人 / 技术管理者。

- 反问面试官

  eg:

  1. （面试比较好的情况下）能不能点评一下我今天的面试，如果要涉入这个岗位的话，您觉得我哪一块知识是不够全面的？
  2. 咱们公司对新人入职之后会有什么指导和培养？因为学校里学习的东西还是跟公司有些许差异的。

基于多设计模式的抽奖系统

技术栈：SpringBoot + SpringMail + MyBatisPlus + MySQL + Redis + RabbitMQ + Vue 3

项目描述：运用责任链、策略、模板方法等多种设计模式构建的抽奖平台，实现活动状态流转、异步抽奖处理、数据加密等核心能力。支持活动创建、奖品管理及中奖通知全流程，服务于营销活动场景。

主要工作：

- 策略模式 + 责任链模式：状态转换模块抽象为多个 Operator，按序编排执行，扩展性强
- 异步抽奖：RabbitMQ 异步解耦抽奖逻辑，支持死信队列容错，MQ 不可用时自动降级为同步处理

设计模式：

- 模板方法模式：抽象操作器定义统一执行模板
- 全局异常处理：统一异常捕获与响应格式

利用AI工具开发的代码仓库搜索引擎

技术栈：SpringBoot + PyTorch + MyBatis-Plus + FAISS + Jieba + 中文向量模型

项目描述：这是一个融合关键词倒排索引与语义向量检索的代码仓库搜索引擎，通过对接 Gitee API 实现仓库数据自动同步，基于 Spring Boot 与 PyTorch 双服务架构提供中文代码文档的全文检索能力。

项目亮点：

1. 实现混合检索策略提升搜索结果质量 — 结合倒排索引（Jieba 分词 + TF-IDF 词频排序）与语义向量检索，通过结果交集融合，兼顾精确匹配与语义相关性
2. 构建中文倒排索引 — 基于 Jieba 搜索引擎模式分词实现内存倒排索引，支持自定义停用词过滤，按全局词频计算相关性得分并排序返回
3. 搭建语义向量检索服务 — 基于 PyTorch 加载中文向量模型生成文本向量，使用 FAISS 进行相似度检索，通过 HTTP 接口与 Java 后端交互
4. 开发 Gitee 仓库数据同步模块 — 对接 Gitee API 实现分页获取用户/组织仓库列表，自动拉取 README 内容并 Base64 解码入库
5. 实现索引持久化与快速恢复 — 倒排索引支持序列化到文件，应用启动时从磁盘加载，避免重复构建索引的开销

  技术细节：

- 向量索引采用 L2 距离的 Flat 索引结构，维度 768
- 支持 CUDA 自动检测与 CPU 降级
- 批量向量推理，批大小 32
- 分页查询 MySQL 数据构建索引（每页 1000 条）
